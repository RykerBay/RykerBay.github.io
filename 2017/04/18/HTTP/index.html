HTTP & HTTPS

HTTP协议

HTTP/1

Connection

  • non-persistent Connection : 一次request就建立一次TCP（三次握手）连接
  • persistent Connection: 保持第一次的TCP连接，之后传信息都是通过这个连接

HTTP本身是无状态的连接（non-persistent Connection），可以通过设置头部的connection属性为keep-alive和设立session ID来保持连接，现在默认是是有状态连接了

Message

Struct

  • HTTP 报文大致可分为首部和主体两块。两者由最初出现的空行(CR+LF)来划分。
  • 首部字段组成类型
  • 通用part （Cache-Control、 Connection、Date、Pragma、Transfer-Encoding、Upgrade、Via）
  • 请求part (Method、Request-URI 、HTTP-Version、Accept、Accept-Charset、Accept- Encoding、Accept-Language、Authorization、From、Host、If-Modified-Since、If- Match)
  • 响应part （HTTP-Version、Status-Code、Reason-Phrase、Age、Location、Proxy-Authenticate、Public、Retry- After、Server、Vary、Warning、WWW-Authenticate）
  • 实体part （Content-Type、Allow、Content- Base、Content-Encoding、Content-Language、Content-Length、Content-Location、Content-MD5）

Proxy Server 代理服务器（Web caching）

目的

存储子Clients访问时server时返回的response信息，当clients再次访问时，就会先访问Proxy Server里的cache，里面没有的话Proxy Server再去向原始目标服务器进行请求

  • 减短Clients得到response的时间
  • 避免网络拥塞（例如：在大型局域网中（校园网），如果差不多时间有很多子Clients同时方位外界的一个Server时，可以高效的返回response）

更新

由于response信息有时效性，所以Proxy Server需要时时判断是否有新信息。当Proxy Server想Server请求的时候，通过当前cache版本的时间（放入last-modified属性中）来让Server判断是否有新的内容，有的话就将最新内容返回并设立新的cache版本的时间

CDN (Content Distribution Network)

通过部署在各地边缘的Server，让Clients就近获取内容，降低网络拥堵

HTTP/2

HTTP/2 vs HTTP/1

新特性

  • 新的二进制格式（Binary Format），HTTP1.x的解析是基于文本。基于文本协议的格式解析存在天然缺陷，文本的表现形式有多样性，要做到健壮性考虑的场景必然很多，二进制则不同，只认0和1的组合。基于这种考虑HTTP2.0的协议解析决定采用二进制格式，实现方便且健壮。

  • 多路复用（MultiPlexing），即连接共享，即每一个request都是是用作连接共享机制的。一个request对应一个id，这样一个连接上可以有多个request，每个连接的request可以随机的混杂在一起，接收方可以根据request的 id将request再归属到各自不同的服务端请求里面。多路复用原理图：
    **

  • Header压缩，如上文中所言，对前面提到过HTTP1.x的header带有大量信息，而且每次都要重复发送，HTTP2.0使用encoder来减少需要传输的header大小，通讯双方各自cache一份header fields表，既避免了重复header的传输，又减小了需要传输的大小。

  • 服务端推送（server push），同SPDY一样，HTTP2.0也具有server push功能。目前，有大多数网站已经启用HTTP2.0，例如YouTuBe，淘宝网等网站，利用chrome控制台可以查看是否启用H2.

图解

HTTP/2 vs SPDY

顾名思义有了HTTP1.x，那么HTTP2.0也就顺理成章的出现了。HTTP2.0可以说是SPDY的升级版（其实原本也是基于SPDY设计的），但是，HTTP2.0 跟 SPDY 仍有不同的地方，主要是以下两点：
HTTP2.0 支持明文 HTTP 传输，而 SPDY 强制使用 HTTPS
HTTP2.0 消息头的压缩算法采用 HPACK，而非 SPDY 采用的 DEFLATE

HTTPS

HTTP Security/ HTTP on TLS

HTTPS是承载在TLS/SSL之上的HTTP，相较于HTTP明文数据传输方面所暴露出的缺点，HTTPS具有防止信息被窃听、篡改、劫持，提供信息加密，完整性校验及身份验证等优势。

HTTPS vs HTTP

  • HTTPS协议需要到CA申请证书，一般免费证书很少，需要交费。
  • HTTP协议运行在TCP之上，所有传输的内容都是明文，HTTPS运行在SSL/TLS之上，SSL/TLS运行在TCP之上，所有传输的内容都经过加密的。建立连接顺序：TCP -> TLS -> HTTP
  • HTTP和HTTPS使用的是完全不同的连接方式，用的端口也不一样，前者是80，后者是443。
  • HTTPS可以有效的防止运营商劫持，解决了防劫持的一个大问题。
